Data engineering, which involves building and maintaining data pipelines, is important because it ensures that we can have good data to work with for things like data analysis and data science. Data engineering is what transforms the manual act of grabbing data from an online file into an automated, scalable, and efficient process for organizations. There are many components that go into a modern data stack and the exact parts and choices would depend on your organization, but below is a simple example. It's important to note that there is a lot of overlap in many modern platforms as they are designed to be multi-functional. For example, you can probably store, transform, and automate workflows using just one tool. Depending on the team, you may only be responsible for part of the data pipeline. As an example, imagine if we want to update a custom summary table with new data about fishing catches on a routine basis so that we can use the table to auto populate BI dashboards and monitor fishing activity. To keep things simple, let's just use a static CSV file as our data source. After downloading the file, we can easily import our data into Databricks and then quickly use a SELECT statement to check that it is there.

Next, we can create our summary table using CREATE TABLE and populate it with a few fields that we care about. Here, you can also do whatever data transformation you'd like. Although I prefer transforming data as upstream as possible for general reusability and consistency, you can also do that further downstream in the BI tool as well, depending on your goals.

On a monthly basis, we can now INSERT into our new table with data from the previous month. For the sake of this example, we are using static values but in reality you would use dynamic values (commented out) for the month and year so that it will automatically work.

To confirm that our data loaded properly, we can use SELECT again, this time grabbing from our summary table.

Now we can easily schedule a job that runs on the first day of the month and updates our summary table with new data.

Finally, we can connect our Databricks table to a BI tool like Power BI and see the data from our custom summary table. The end result is an automated segment of the data pipeline that updates a custom table with new data on a routine basis. The data is then pulled by a BI tool which, depending on the storage mode you choose, will automatically refresh with new data on a routine basis as well.
